04-驗證登入及 Vue Router 的配置

Posted on 2021-04-10 In vue Disqus:

上一章我們完成了登入及登出，這一章節主要是要藉由 API 來檢查使用者登入狀態，讓已登入者跳轉路由，而不讓未登入者進入，另外在使用者任意輸入網址時，直接導入登入頁。

相關資源

導航守衛| Vue Router
https://router.vuejs.org/zh/guide/advanced/navigation-guards.html

步驟備忘

先將導航守衛的代碼貼入 main.js ，然後將狀態印出來。（課程說要換頁才能作用，但我發現一載入頁面就能發揮效果，可能 Vue 已有更新）

守衛方法有三個參數，分別是：

  1. to：要前往的路由
  2. from：前一個路由
  3. next：這是一個方法，允許前往下一個路由。裡面可以包一個路由路徑，例如

為原本的路由加入 requiresAuth 屬性，以讓導航守衛得以判斷。

加入判斷，如果該頁需要驗證，才執行 API 檢查，否則直接進入。

到這邊基本上就完成了。如果使用者輸入任意網址會看到空白頁，我們要讓在這時候讓導入路由 Login。
